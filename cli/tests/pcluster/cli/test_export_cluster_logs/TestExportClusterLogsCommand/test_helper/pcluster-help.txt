usage: pcluster export-cluster-logs [-h] --cluster-name CLUSTER_NAME
                                    [--region REGION] --bucket BUCKET
                                    [--bucket-prefix BUCKET_PREFIX]
                                    [--start-time START_TIME]
                                    [--end-time END_TIME]
                                    [--filters FILTERS [FILTERS ...]]
                                    [--debug] [--query QUERY]

Export the logs and stack events for a given cluster.

optional arguments:
  -h, --help            show this help message and exit
  --cluster-name CLUSTER_NAME
                        Name of the cluster
  --region REGION       Region that the given cluster and bucket belong to.
  --bucket BUCKET       S3 bucket to export cluster logs data to. It must be
                        in the same region of the cluster.
  --bucket-prefix BUCKET_PREFIX
                        Keypath under which exported logs data will be stored
                        in s3 bucket.
  --start-time START_TIME
                        Start time of interval of interest for log events. ISO
                        8601 format: YYYY-MM-DDThh:mm:ss.sssZ (e.g.
                        1984-09-15T19:20:30.000Z), time elements might be
                        omitted. Defaults to creation time.
  --end-time END_TIME   Start time of interval of interest for log events. ISO
                        8601 format: YYYY-MM-DDThh:mm:ss.sssZ (e.g.
                        1984-09-15T19:20:30.000Z), time elements might be
                        omitted. Defaults to current time.
  --filters FILTERS [FILTERS ...]
                        Filter the log streams. Format: (Name=a,Values=1
                        Name=b,Values=2,3). Accepted filters are: private-dns-
                        name - The short form of the private DNS name of the
                        instance (e.g. ip-10-0-0-101). node-type - The node
                        type, the only accepted value for this filter is
                        HeadNode.
  --debug               Turn on debug logging.
  --query QUERY         JMESPath query to perform on output.
